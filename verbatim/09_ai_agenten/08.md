## Beschreibung

\subsection{Code-Review-Agent}
Ein SaaS-Unternehmen nutzt einen Multi-Agent-Workflow für automatische Code-Reviews:

## Bildbeschreibung

Supervisor-Worker-Architektur: Oben Eingabe 'PR geöffnet' als Kästchen. Pfeil zum Supervisor-Kästchen (zentral, grün hervorgehoben). Vom Supervisor drei horizontale Pfeile zu drei Worker-Kästchen: Security (rot), Quality (blau), Performance (orange). Antwortpfeile zurück zum Supervisor. Dieser aggregiert und leitet an 'Reviewer' (unten) weiter, der das 'Feedback im PR' ausgibt. Klare Hierarchie: oben → mitte → unten. Modernes Flussdiagramm.

## Verbatim

```
   [PR geoeffnet] -> [Supervisor]
                       |
           +-----------+-----------+
           |           |           |
       [Security]  [Quality]  [Performance]
           |           |           |
           +-----------+-----------+
                       |
                   [Reviewer]
                       |
                   [Feedback im PR]
```
