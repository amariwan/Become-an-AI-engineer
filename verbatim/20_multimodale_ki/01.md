# 20 multimodale ki – Block 1

## Beschreibung

\subsection{Modalitäten und ihre Verarbeitung}
Ein multimodales Modell unterscheidet sich fundamental von einem Text-LLM: Es besitzt separate Encoder für jede Modalität (Vision, Audio, Text), die in einen gemeinsamen Representierungsraum projiziert werden. Die Entscheidungsebene (Transformer-Decoder) operiert auf dieser fusionierten Repräsentation.

## Bildbeschreibung

Multimodale Architektur: Drei Input-Pfade von links – Text (blau), Image (grün), Audio (orange). Jeder führt in seinen Encoder (Tokenizer, Vision Encoder, Audio Encoder). Alle Encoder münden in einen zentralen Fusion Encoder (violett). Vom Fusion Encoder Pfeil zum Decoder/LLM (rechts, grosses Kästchen). Moderner Datenfluss von links nach rechts, farblich kodierte Modalitäten. Für Buch geeignet: sauber, erklärend, hierarchisch.

## Verbatim

```
   +--------+    +-----------+    +---------+
   | Text   |--->| Tokenizer |--->|         |
   +--------+    +-----------+    |         |
   +--------+    +-----------+    | Fusion  |    +----------+
   | Image  |--->| Vision    |--->| Encoder |--->| Decoder  |
   +--------+    | Encoder   |    |         |    | (LLM)    |
   +--------+    +-----------+    |         |    +----------+
   | Audio  |--->| Audio     |--->|         |
   +--------+    | Encoder   |    +---------+
                  +-----------+
```
