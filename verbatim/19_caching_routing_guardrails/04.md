\section{Guardrails -- Sicherheit für LLM-Systeme}
Guardrails sind Validierungsschichten vor und nach dem LLM-Call:

## Bildbeschreibung

Guardrail-Pipeline von links nach rechts: User Input → Input Guard (mit drei Prüfungen: PII Redact, Injection Check, Length Limit) → LLM → Output Guard (Content Filter, Schema Validate, Hallucination Check) → User. Guard-Kästchen farblich hervorgehoben (rot für Input, orange für Output), LLM-Kästchen blau. Prüfungen als untergeordnete Liste in den Guard-Kästchen. Pfeile verbinden die Stationen. Modern, klar, bedrohungsorientiert.

## Verbatim

```
   [User Input] --> [Input Guard] --> [LLM] --> [Output Guard] --> [User]
                        |                           |
                   [PII Redact]              [Content Filter]
                   [Injection Check]          [Schema Validate]
                   [Length Limit]             [Hallucination Check]
```
