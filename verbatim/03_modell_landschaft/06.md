# 03 modell landschaft – Block 6

## Beschreibung

\section{Architektur -- Multi-Provider-Routing}
Ein produktives System verlässt sich nie auf einen einzigen Modell-Provider. Die folgende Architektur zeigt ein Gateway, das Anfragen je nach Anforderung an verschiedene Modelle routet:

## Bildbeschreibung

Intent-Routing-Baum: Oben ein User-Query-Kästchen. Pfeil nach unten zum Intent-Classifier (Rautenform). Drei Verzweigungen: links 'Simple Q' → kleines, günstiges Modell, Mitte 'Complex Q' → grosses Modell, rechts 'Code Q' → Code-spezifisches Modell. Jeder Pfad farblich codiert (grün/gelb/blau). Kosten- und Latenz-Angaben pro Pfad als kleine Labels. Modernes Baum-Layout von oben nach unten.
